# macOS 微信聊天记录导出 HTML：从逆向 SQLCipher 加密到完整工具链，开源了

做这个的起因很简单——我想把自己和某个人几年的聊天记录导出来保存，结果发现 macOS 版微信没有任何导出功能。网上大部分方案是 Windows 版的，Mac 这边要么要付费，要么原理讲不清楚。于是花了两天逆向，自己写了一套完整工具链，开源出来，希望能帮到有同样需求的人。

## 效果先看

导出的 HTML 长这样，浏览器直接打开就能看：

- 顶部绿色导航栏，显示私聊/群聊数量和消息总数
- 可点击目录，按最近消息排序
- 消息列表：发送者 + 时间 + 内容，绿色是自己，黑色是对方
- 群聊里不同人用不同颜色区分
- 图片/语音/表情标注类型，系统消息灰色显示

我自己的数据：4621 个联系人，230 个会话（122 私聊 + 44 群聊），172898 条消息，导出约 100MB HTML。

单个联系人的聊天导出只有几百 KB，打开秒开。

## 原理（说人话版）

macOS 微信把聊天记录存在 SQLite 数据库里，但加了 SQLCipher 加密——每个 .db 文件有独立的 AES-256 密钥，没有统一主密钥。

所以要导出数据只需要做三件事：

1. **拿到密钥** — 微信在打开数据库时要经过苹果的 CommonCrypto 库，用 Frida Hook 这个库的 `CCCryptorCreate` 函数，就能在密钥出现的那一刻截获它
2. **解密数据库** — 拿到密钥后用 sqlcipher 命令行工具把加密库转成明文 SQLite
3. **解析导出** — 读明文数据库，搞清楚表结构（联系人/会话/消息怎么存的），生成 HTML

## 技术细节

### 为什么每个数据库有独立密钥？

微信的数据库分布在 `contact/`、`message/`、`session/` 等目录下，一共十几个 .db 文件。每个都有不同的 AES-256 密钥。如果用统一密钥，任意一个库被破解全部泄漏。独立密钥意味着你需要 Hook 捕获十几个不同的密钥才能解密全部数据——这本身也是一种安全设计。

### 解密本地文件 vs 抓包

很多人听到"导出微信数据"本能想到网络抓包。但聊天记录不在网络上——微信已经从服务器同步到本地了，加密存在 SQLite 文件里。我们做的不是拦截通信，而是**解密自己电脑上已有的文件**。从头到尾没有碰微信服务器，也没有截获任何网络流量。

### 不同数据库里"我"的内部 ID 不一样

这是解析阶段踩的最大的坑。同一个人的消息，在 `message_0.db` 里 `real_sender_id` 可能是 2，到了 `message_1.db` 里变成 6。但对方联系人的内部 ID 保持不变。利用这个规律，可以通过交叉比对推断用户在不同数据库中的 ID。

### 群聊消息格式和私聊完全不同

- 私聊：`message_content` 是纯文本，不含发送者信息
- 群聊：`message_content` 格式是 `wxid_xxx:\n消息内容`，发送者直接写在前面

如果有人在群里只发图片不发文字，就需要通过 `real_sender_id` 映射来推断。

## 使用方法

```bash
# 1. 安装依赖
pip install frida-tools
brew install sqlcipher

# 2. Hook 微信捕获密钥
frida -n WeChat -l hook_keys.js
# 在微信里多点点不同的聊天、联系人、朋友圈，触发更多密钥

# 3. 验证密钥 → 填入 decrypt_all.py → 解密
python3 decrypt_all.py

# 4. 导出全部聊天为 HTML
python3 export_all.py

# 或者导出单个联系人
# 修改 export_single.py 里的备注名 → 运行
python3 export_single.py
```
